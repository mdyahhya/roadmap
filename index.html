How You Can Do It

Stop Studying. Start Building.

Real growth as a developer doesn't come from watching 40-hour courses or collecting certificates. It comes from sitting down, picking a problem, and figuring it out. Every project you ship — even a messy one — teaches you more than any tutorial. Start in your 2nd or 3rd year. Build something. Break it. Fix it. Repeat. That cycle is the entire career.

🔨

Build First

Pick any idea, no matter how small. A to-do app, a personal site, a calculator. Shipping is learning.

🔁

Iterate Constantly

Your first version will be bad. That's fine. Come back, improve it, add features, refactor the code.

🌍

Solve Real Problems

The best projects fix something real — for you, your family, a local business. Reality beats imagination.

Reality Check

Common Mistakes That Kill Careers Early

Most students fall into the same traps. Recognising them is half the battle.

📜

Certificate Chasing

Certificates feel productive but rarely impress hiring teams. Projects and GitHub repos matter far more.

🎥

Tutorial Hell

Watching a 10-hour React course then starting another one without ever building anything real is the trap.

🚫

Zero Projects

Many graduates have a degree but nothing to show a recruiter. Your GitHub is your real CV.

⏳

Waiting to Feel Ready

You will never feel fully ready. Start broken, ship anyway, learn as you go.

🔒

Single Domain Trap

Only doing websites limits you. Explore apps, AI tools, automation, desktop software — the world is wider.

🤖

Blind AI Copy-Paste

AI is powerful, but copying code you don't understand is a debt that catches up fast. Use AI to learn, not just do.

Skills Roadmap

What to Learn & In What Order

A practical learning path — front to back, not all at once.

Frontend Basics — HTML · CSS · JavaScript

The non-negotiable foundation. Learn semantic HTML structure, CSS layouts (Flexbox, Grid), and JavaScript fundamentals — DOM manipulation, events, fetch API. Everything else builds on top of this.

HTML5CSS3JavaScript ES6+DOMFlexboxGrid

React — Frontend Framework

Once JavaScript clicks, move to React. Learn props, state and more. React is the most in-demand frontend skill for jobs right now.

Backend — Node.js

Build APIs with Node.js. Understand RESTful routes, request/response cycles, and how a server handles data. This is the bridge between your frontend and your database.

Node.jsREST APIJSON

SQL Databases

Learn SQL — structured queries, table relationships, and database design are essential for real-world systems. Many company databases are SQL-based.

SQLSupabasePostgreSQLLive Databases

Python — Scripting, Libraries & AI

Python is powerful for automation, data processing, and AI development. Start with the basics, then explore libraries like NumPy, Pandas, and frameworks for AI. Essential if you want to build intelligent tools and custom AI applications.

PythonLibrariesNumPyPandasFastAPIAI/ML

Deploy & Backend

Your First Live Project & Real Databases

Making something live on the internet is a milestone. Then connect it to a real database.

01

Write Your Code

Build your project locally. Make sure your entry point is index.html for static sites or the right server file for backends.

02

Push to GitHub

Create a repo, commit your code, and push. GitHub is not just storage — it's your professional portfolio visible to every recruiter.

03

Host on Netlify / Vercel

Connect your GitHub repo to Netlify or Vercel. Get a free live URL in minutes. Every future commit auto-deploys. This is the professional workflow.

04

Supabase for Databases

Use Supabase to create live SQL databases without a backend server. Connect using API keys, manage tables visually, and build full-stack apps fast — even use AI to write your SQL queries.

🔑

API Keys

Understand what API keys are, how to store them securely (env variables), and never commit them to GitHub.

🔗

Frontend ↔ Backend

Learn how fetch() and axios connect your UI to a backend. Understand CORS, JSON responses, and loading states.

🗄️

Database Tables

Design simple schemas. Users, products, orders. Understand primary keys, foreign keys, and basic CRUD queries.

06 / 09
AI Development

How AI Works & How to Build With It

You don't need a PhD to build with AI. Understanding the basics lets you build real tools that impress.

01 · FOUNDATION

How LLMs Work

Large Language Models predict the next token based on patterns from training. They don't "think" — they pattern-match at scale. Understanding this helps you prompt and use them better.

02 · INTEGRATION

Using AI APIs

OpenAI, Anthropic, Gemini — all expose REST APIs. You send a message, you get a response. That's it. Learn to call these from your JavaScript or Python code with a simple fetch request.

03 · ADVANCED

RAG — Custom AI on Your Data

Retrieval-Augmented Generation (RAG) lets you build chatbots that answer from your own data. Upload your documents, vectorize them, and the AI retrieves relevant context before answering. This is how ai.yahya.in works.

04 · MINDSET

AI as a Tool, Not a Crutch

Use AI to brainstorm, explain concepts, and speed up tasks. Never blindly paste code you can't explain. The developer who understands and uses AI is 10× more powerful than one who only copies from it.

Career

Getting Your First Job

Your CV is your first impression. Your GitHub is the proof. LinkedIn is where opportunities quietly arrive.

📄

ATS-Optimised CV

Most companies run CVs through ATS software first. Use clean formatting, relevant keywords, and lead with your strongest projects. Keep it to one page as a student.

🐙

Active GitHub

Commit regularly. Even small contributions. A green contribution graph signals consistency. Pin your best 6 repos. Write good READMEs — they show communication skills.

🔗

LinkedIn Presence

Post about what you're building. Connect with developers and recruiters. Many real opportunities arrive through DMs from people who noticed your posts. Don't underestimate this platform.

🎯

Start Early — Apply Often

Begin building your CV in 2nd year. Apply even when you don't feel ready. Every rejection is data. Keep your CV constantly updated as you ship new projects.

🌐

Portfolio Website

A live portfolio domain (e.g. yahya.in) is a conversation starter and a trust signal. It shows you can actually build and deploy things — not just claim you can.

🤝

Partner & Freelance

You don't need a job to get experience. Help a local business, build for a friend's startup, or partner on someone else's idea while handling the tech side.
